Web安全-某系统目录遍历引发的数据泄露
发表于 2019-12-23| 更新于 2019-12-23
1、引言

在测试一些复杂的站点的时候由于模块较多，功能丰富，测试起来尤为繁琐，那么当我们遇到类似于电商，政务后台的时候，我们需要把模块进行分解测试，比如，有数据交互的模块我们需要着重测试：登陆模块、忘记密码模块，订单提交模块、地址添加模块，数据导出模块等，凡是关键参数没有进行加密的要进行重点测试，每个模块所用到的方法，还需要我们自己去了解学习，从而应用到实际的渗透中去。

2、实例

偶遇一个系统在测试时未发现，诸如一些比较高危的逻辑漏洞和信息泄露，服务器开放的端口也很少，跑了下C段，也没有太大收获。（我在测试的时候一般遵循的流程步骤：先进行信息收集找高危类似于信息泄露/逻辑漏洞/SSRF/等–其次是扫描服务器开放的端口，查看服务进行利用–然后进行C段扫描和基础漏洞扫描，这样做的原因是避免一开始就进行扫描，防止被对方封禁，在做业务的时候，也是如此，无法访问，将会导致测试工作无法展开。）在大部分方法都已经尝试过之后，我决定对目录进行爆破。

4.总结

我们在作渗透测试的时候一定不要被自己的脑回路限定，多找其他思路和方法，只有不断摸索，不断尝试，才能提升自己，黑盒测试一切都是未知的，敢想（当然是合理的情况下哈）敢尝试，才能收获结果。
