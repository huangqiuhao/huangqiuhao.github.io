主机安全-Win Server 系列安全检查项
发表于 2019-07-26| 更新于 2019-08-14|系统安全
NO 检查类别 检查项目 检查要点 检查对象 检查方式 检查方式 检查方法
1 账号管理 检测系统自带的与设备运行、维护等工作无关的账号是否被删除或锁定 对于管理员帐号，要求更改缺省帐户名称；禁用guest（来宾）帐号;删除无用账户。 Windows 2003、2008、2012 检查项 进入“控制面板->管理工具->计算机管理”，在“系统工具->本地用户和组”： 缺省帐户Administrator－>属性 Guest帐号->属性 符合： administrator帐号改名 guest帐号禁用 不存在其他帐号或存在的帐号有明确的说明 不符合： administrator帐号未改名且未说明原因 guest帐号未禁用 存在其他帐号且没有明确的说明
2 检查是否设置帐户定时自动登出 对于远程登录的账户，设置不活动所连接时间15分钟 Windows 2003、2008、2012 检查项 开始–>运行–>gpedit.msc 计算机配置->管理模板->Windows组件->终端服务->会话 为断开的会话设置时间限制 已启用 到达时间限制时终止会话 已启用 符合：两项均不高于15分钟 不符合：两项中有高于15分钟的
3 口令管理 密码安全要求 1、口令中不能含有与IP地址、账号名、公司名、人名、生日、记念日期、电话、E-mail等有关的字符； 2、不同主机、不同账号均不能使用同一个密码； 3、不同主机、不同账号均不能使用有规律的密码； Windows 2003、2008、2012 检查项 1、弱口令检查步骤一： 1）检查用户密码是否存在弱密码（重点检查用户名与密码相同的极弱密码）（注：需要导出配置文件、离线扫描） 2、弱口令检查步骤二： 1）省公司将4A系统级所有资源的口令密文保存为pass.txt，放置在4A服务器中。注：密码用回车符分隔 2）省公司将检查组提供的“弱口令字典”和“密码比对脚本”拷贝至同一台4A服务器； 3）省公司使用4A系统的口令加密方式，将弱口令字典转换为”弱口令密文字典” 4）运行密码比对脚本。如果”4A密文口令”与”弱口令密文字典”中存在重复的密码，则本项不通过。 密码破解成功则认为不符合
4 检查用户口令策略是否符合安全要求 最短密码长度8个字符，启用本机组策略中密码必须符合复杂性要求的策略。即密码至少包含以下四种类别的字符中的三种： l 英语大写字母 A, B, C, … Z l 英语小写字母 a, b, c, … z l 西方阿拉伯数字 0, 1, 2, … 9 非字母数字字符，如标点符号，@, #, $, %, &, *等 对于采用静态口令认证技术的设备，账户口令的生存期不长于90天。 对于采用静态口令认证技术的设备，应配置设备，使用户不能重复使用最近6次（含6次）内已使用的口令。 在域环境下域服务器安全策略优先级高于本地安全策略议，检查域服务安全策略设置。 Windows 2003、2008、2012 检查项 1、进入“控制面板->管理工具->本地安全策略”，在“帐户策略->密码策略”： 查看是否“密码必须符合复杂性要求”选择“已启动” 2、进入“控制面板->管理工具->本地安全策略”，在“帐户策略->密码策略”： 查看“密码最长使用期限” 3、进入“控制面板->管理工具->本地安全策略”，在“帐户策略->密码策略”： 查看“强制密码历史” 符合： “密码必须符合复杂性要求”选择“已启动” “密码最长存留期”不长于90天 “强制密码历史”大于等于5 不符合：未按照上述策略配置
5 补丁漏洞管理 检查补丁更新情况 检查WINDOWS系统的SERVICE PACK是否符合要求，并及时更新微软发布的中高危补丁。安装的SP如下： 1、WINDOWS 2003：SP2 2、WINDOWS 2008：SP2 3、WINDOWS 2008 R2：SP1 Windows 2003、2008、2012 检查项 查看系统版本 1.在“开始->运行->键入cmd”，运行systeminfo，查看“OS 版本”是否为要求版本 符合： 1、WINDOWS 2003为SP2版本 2、WINDOWS 2008为SP2版本 3、WINDOWS 2008 R2为 SP1版本 未发现中高危操作系统漏洞 不符合 1、Windows 2003 低于SP2版本 2、Windows 2008 低于SP2版本 3、Windows 2008 R2低于SP1版本 发现中高危操作系统漏洞
6 漏洞管理 对于中高危漏洞，应及时修复；对于暂无修复手段的漏洞，应通过其他技术手段进行风险规避； 注：中高危漏洞以国内外权威的CVE漏洞库和国家互联网应急中心CNVD漏洞库最新版为判断依据。 Windows 2003、2008、2012 检查项 1.对目标主机进行漏洞扫描，要求扫描后未发现中高危操作系统漏洞 符合： 未发现中高危操作系统漏洞 不符合 发现中高危操作系统漏洞
7 系统服务 检查WINDOWS主机是否开启了不必要的服务 对系统运行的服务应该进行严格控制，一些不应该开启的服务或是会带来安全性问题的服务原则上都应该关闭。 停止如下不需要的服务： ü DHCP server services ü WINS Service ü Fax Service Bluetooth Support Service ü Indexing Services ü Messenger Service ü Print Spooler ü Remote Registry Service（AD域环境除外） Server（AD域环境除外） ü Simple TCP/IP Services Wireless Zero Configuration SMTP/FTP/IIS Admin/WWW Publish SNMP Community String （如果需要IIS服务，可关闭由IIS提供的SMTP、FTP服务） 按照服务最小化原则，禁用所有不需要的服务。 Windows 2003、2008、2012 检查项 1、在“开始->运行->键入services.msc”，确认在服务中查看相应的服务已经关闭。 2、打开“控制面板”，打开“管理工具”中的“服务”，找到“SNMP Service”，单击右键打开“属性”面板中的“安全”选项卡，在这个配置界面中，可以修改community strings（团体名称）。 备注：若未开启SNMP服务，则此项符合要求。 符合： 以下服务停用 DHCP server services ü WINS Service ü Fax Service Bluetooth Support Service ü Indexing Services ü Messenger Service ü Print Spooler ü Remote Registry Service（AD域环境除外） Server（AD域环境除外） ü Simple TCP/IP Services Wireless Zero Configuration SMTP/FTP/IIS Admin/WWW Publish 且未停用服务提交了原因说明 不符合：以上服务未停用或未停用服务没有提交原因说明
8 安全配置 检查屏幕保护程序安全配置 启用屏幕保护程序，防止管理员忘记锁定机器被非法攻击；设置带密码的屏幕保护，并将时间设定为5分钟以内 Windows 2003、2008、2012 检查项 进入“控制面板－>显示－>屏幕保护程序”： 启用屏幕保护程序，设置等待时间为“5分钟”，启用“在恢复时使用密码保护”。 备注：设置屏保不大于5分钟即为满足 符合： 启用屏幕保护程序，等待时间小于等于5分钟 且 启用“在恢复时使用密码保护” 不符合： 未启用屏幕保护程序或等待时间大于5分钟 或 未启用“在恢复时使用密码保护”
9 检查是否安装防病毒软件 检查windows设备是否安装了统一部署的防病毒软件 Windows 2003、2008、2012 检查项 控制面板->添加或删除程序，是否安装有防病毒软件。打开防病毒软件控制面板，查看病毒码更新日期。 符合： 安装了公司统一部署的防病毒软件 且 更新时间距最新发布不超过7天 不符合： 未安装公司统一部署的防病毒软件 或更新时间距最新发布超过7天
10 RDP协议 1.RDP访问协议加密 2.RDP会话超时控制 3.RDP登录设置 Windows 2003、2008、2012 检查项 1.查看，开始->管理工具->终端服务配置->连接->双击打开RDP-Tcp->常规->加密级别 2.查看，开始->管理工具->终端服务配置->连接->双击打开RDP-Tcp->会话->”结束已断开的会话”和”空闲会话限制” 3.查看，开始->管理工具->终端服务配置->连接->双击打开RDP-Tcp->登录设置 符合： 1.加密级别为“高” 2.两项均不高于15分钟 3.勾选了“总是提示密码” 不符合： 1.加密级别低于“高” 2.两项中有高于15分钟的 3.没有勾选“总是提示密码”
11 远程注册表 禁用可远程访问的注册表路径和子路径 Windows 2003、2008、2012 检查项 查看“控制面板->管理工具->本地安全策略”，在“本地策略->安全选项”:网络访问中，查看，可远程访问的注册表路径、可远程访问的注册表路径和子路径 符合：可远程访问的注册表路径、可远程访问的注册表路径和子路径内容为空 不符合：可远程访问的注册表路径、可远程访问的注册表路径和子路径内容非空
12 FTP安全 1、控制FTP进程缺省访问权限，当通过FTP服务创建新文件或目录时应屏蔽掉新文件或目录不应有的访问允许权限； 2、主机FTP服务，关闭匿名登录方式； 3、禁止Administrator及拥有系统管理员权限的用户登陆FTP； 4、修改系统FTP banner，避免泄漏FTP名称，版本号，并且给出登陆告警信息 Windows 2003、2008、2012 检查项 “管理工具”→“Internet信息服务（IIS）管理器”菜单项，打开“Internet信息服务（IIS）管理器”窗口。在左窗格中展开“FTP站点”目录，右键单击“默认FTP站点”选项，并选择“属性”，右键单击“默认FTP站点”选项，切换到“主目录”选项卡，单击“浏览”按钮在本地计算机磁盘中选择要作为FTP站点主目录的文件夹，并依次单击“确定”按钮。根据实际需要选中或取消选中“写入”复选框，以确定用户是否能够在FTP站点中写入数据 点击‘安全账户’去掉‘允许匿名连接’选项卡 点击“消息”更改‘标题’、‘欢迎’、‘推出’内容 符合：定义了目标目录，关闭匿名登录选项卡；禁止Administrator帐号登录；更改banner信息。 不符合：未定义了目标目录，匿名登录选项卡打勾；Administrator帐号可以登录；未更改banner信息。
13 信息泄露 需对rpc、DCE/RPC服务枚举漏洞 Windows 2003、2008、2012 检查项 使用netstat -antpo命令查看开放端口以及对应的进程信息 符合：不存在相关开放端口 不符合：存在相关开放端口
14 检查是否开启防火墙 启用自带防火墙或安装第三方威胁防护软件。根据业务需要限定允许访问网络的应用程序，和允许远程登陆该设备的IP地址范围 Windows 2003、2008、2012 检查项 进入“控制面板－>网络连接－>本地连接－>属性”，在高级选项的设置中查看是否启用了Windows防火墙 在“例外”中查看列出的程序和服务 启用Windows防火墙。“例外”中允许接入网络的程序均为业务所需。
15 NTFS格式 检查磁盘分区格式是否为NTFS格式 Windows 2003、2008、2012 检查项 在资源管理器中右键点击磁盘->属性->常规，查看文件系统格式 符合：所有磁盘分区均为NTFS格式 不符合：存在FAT格式的磁盘分区
16 访问控制 检查Windows系统的共享是否禁用； 检查有无自建的共享目录。 非域环境中，检查系统的默认共享是否开启。域环境下可以有必须的共享SYSVOL 和 NETLOGON等。 备注： 系统的默认共享,如：C$、D$； 用户自建共享，指用户自己创建的共享目录 Windows 2003、2008、2012 检查项 在cmd下输入#net share查看共享情况。 符合：net share查看没有共享文件 不符合：net share查看到共享文件
17 检查远程连接工具配置安全 检查WINDOWS系统的远程桌面是否开启。如果开启，检查是否MS12-020等补丁是否安装。 Windows 2003、2008、2012 检查项 1。读取注册表，HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server下的fDenyTSConnections（dword类型）值设为0，不为0，判断远程桌面是否打开，没有打开则通过 2。如果远程桌面打开了，在“开始->运行->键入cmd”，再键入”systeminfo”，在回显信息中查看是否有安装KB2621440安全更新 C:>systeminfo | find “2621440” [222]: KB2621440 - Update在输出中检查是否包含2621440补丁，如果没有则失败。 符合：没有开启远程桌面或者安装了KB2621440安全更新 不符合：开启远程桌面且未安装安装KB2621440安全更新
18 检查远程关机配置安全 在本地安全设置中从远端系统强制关机只指派给Administrators组。 Windows 2003、2008、2012 检查项 进入“控制面板->管理工具->本地安全策略”，在“本地策略->“用户权利指派”或“用户权利分配”: 查看“从远端系统强制关机”设置 符合：“从远端系统强制关机”仅配置为Administrators组 不符合：“从远端系统强制关机”配置除Administrators组外还有其他用户组
19 检查本地关机配置安全 在本地安全设置中关闭系统仅指派给Administrators组。 Windows 2003、2008、2012 检查项 进入“控制面板->管理工具->本地安全策略”，在“本地策略->“用户权利指派”或“用户权利分配”: 查看“关闭系统”设置 符合：“关闭系统”仅配置为Administrators组 不符合：“关闭系统”配置除Administrators组外还有其他用户组
20 检查用户权利指派配置安全 在本地安全设置中取得文件或其它对象的所有权仅指派给Administrators。 Windows 2003、2008、2012 检查项 进入“控制面板->管理工具->本地安全策略”，在“本地策略->“用户权利指派”或“用户权利分配”： 查看是否“取得文件或其它对象的所有权”设置 符合：“用户权利指派”或“用户权利分配”仅指派给Administrators 不符合：“用户权利指派”或“用户权利分配”除指派给Administrators外还有其他用户、用户组
21 远程管理地址安全要求 对于通过IP协议进行远程维护的设备，设备应支持对允许登陆到该设备的IP地址范围进行设定； Windows 2003、2008、2012 检查项 打开“管理工具”—“本地安全设置”，点击“IP安全策略 在本地机器”选择“创建IP策略”，“新IP安全策略 属性”，选择“寻址”标签，源地址设为‘任何IP地址’，目的地址设为‘管理终端IP’—“确定” 符合：设置了管理源IP地址 不符合：未设置管理源IP地址
22 禁止不安全的管理方式 禁止使用telnet远程管理； Windows 2003、2008、2012 检查项 cmd中输入services.msc，找到Telnet服务–属性–将启动类型改为已禁用，服务状态改变为“停止” 符合：Telnet服务–属性–将启动类型为启用 不符合：Telnet服务–属性–将启动类型改为已禁用，服务状态改变为“停止”
23 日志审计 检查日志文件大小设置是否合理 查看是否“应用程序日志”、“安全日志”、“系统日志”属性中的日志大小设置不小于“8192KB” ,设置当达到最大的日志尺寸时，“按需要改写事件” Windows 2003、2008、2012 自查项 进入“控制面板->管理工具->事件查看器”，在“事件查看器（本地）”中： 查看是否“应用日志”属性中的日志大小设置不小于“51200KB” ,设置当达到最大的日志尺寸时，“按需要改写事件” 符合：登陆SMP或日志分析服务器，可查看主机相关日志 不符合：登陆SMP或日志分析服务器，不能查看主机相关日志
24 检查审核登录是否配置 设备应配置日志功能，对用户登录进行记录，记录内容包括用户登录使用的账号，登录是否成功，登录时间，以及远程登录时，用户使用的IP地址。 Windows 2003、2008、2012 自查项 开始->运行-> 执行“ 控制面板->管理工具->本地安全策略->审核策略” 审核登录事件。 备注：设置为成功或失败或两项均审核即符合要求。 符合：审核登录事件设置为成功、失败两项均审核 不符合：审核登录事件设置成功或失败其中一项未审核
25 检查审核策略更改配置 启用组策略中对Windows系统的审核策略更改 Windows 2003、2008、2012 自查项 进入“控制面板->管理工具->本地安全策略”，在“本地策略->审核策略”中 查看“审核策略更改”设置。 备注：设置为成功或失败或两项均审核即符合要求。 符合：“审核策略更改”设置为成功、失败两项均审核 不符合“审核策略更改”设置成功或失败其中一项未审核
26 检查审核对象访问配置 启用组策略中对Windows系统的审核对象访问 Windows 2003、2008、2012 自查项 进入“控制面板->管理工具->本地安全策略”，在“本地策略->审核策略”中： 查看“审核对象访问”设置。 备注：设置为成功或失败或两项均审核即符合要求。 符合：“审核对象访问”设置为成功、失败两项均审核 不符合：“审核对象访问”设置成功或失败其中一项未审核
27 检查审核过程追踪配置 启用组策略中对Windows系统的审核过程追踪 Windows 2003、2008、2012 自查项 进入“控制面板->管理工具->本地安全策略”，在“本地策略->审核策略”中： 查看“审核过程追踪 ”设置。 备注：设置为成功或失败或两项均审核即符合要求。 符合：“审核过程追踪”设置为成功、失败两项均审核 不符合：“审核过程追踪”设置成功或失败其中一项未审核
28 检查审核事件目录服务访问配置 启用组策略中对Windows系统的审核事件目录服务访问 Windows 2003、2008、2012 自查项 进入“控制面板->管理工具->本地安全策略”，在“本地策略->审核策略”中： 查看“审核事件目录服务访问”设置。 备注：设置为成功或失败或两项均审核即符合要求。 符合：“审核事件目录服务访问”设置为成功、失败两项均审核 不符合：“审核事件目录服务访问”设置成功或失败其中一项未审核
