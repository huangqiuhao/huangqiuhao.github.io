详细介绍：如何搭建jumpserver并授权管理大量资产
发表于 2020-03-18| 更新于 2020-03-19

一、简介

jumpserver 是全球首款完全开源的堡垒机, 使用 GNU GPL v2.0 开源协议, 是符合 4A 的专业运维审计系统。使用 Python / Django 进行开发, 遵循 Web 2.0 规范, 配备了业界领先的 Web Terminal 解决方案, 交互界面美观、用户体验好。采纳分布式架构, 支持多机房跨区域部署, 中心节点提供 API, 各机房部署登录节点, 可横向扩展、无并发访问限制。现已支持管理 SSH、 Telnet、 RDP、 VNC 协议资产。

七、权限说明

jumpserver的核心可以说就是用户权限的管理，我们可以通过设置权限，实现免密登陆、手动登陆、自动化推送，权限最小化管理，在这里我们给大家一一列出。权限的最小化原则可以保证我们管理的安全性，很好的在jumpserver后台进行日志记录，安全风险溯源，危险操作管控等。当然我们也要根据自己的实际的网络架构进行部署和操作设置。

用户管理-用户列表：设置登陆后台的用户，比如我给开发组分一个账户，账户内有开发组的服务器，这样开发组登陆jump server后只能看到自己的服务器进行一些列操作。

管理用户：管理用户是资产（被控服务器）上的 root，或拥有 NOPASSWD: ALL sudo 权限的用户， Jumpserver 使用该用户来 推送系统用户、获取资产硬件信息 等。

系统用户: Jumpserver 跳转登录资产时使用的用户，可以理解为登录资产用户，如 web，sa，dba（ssh web@some-host），而不是使用某个用户的用户名跳转登录服务器（ssh xiaoming@some-host）； 简单来说是用户使用自己的用户名登录 Jumpserver，Jumpserver 使用系统用户登录资产。 系统用户创建时，如果选择了自动推送，Jumpserver 会使用 Ansible 自动推送系统用户到资产中，如果资产（交换机）不支持 Ansible，请手动填写账号密码。
